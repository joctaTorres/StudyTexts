Open Systems Interconnection(OSI)
O modelo OSI tem três conceitos fundamentais:
1. Serviços: informa o que a camada faz
2. Interfaces:informa como os processos acima dela podem acessa-la
3. Protocolos: de resposabilidade dessa camada, pode altera-los sem influenciar o software das camadas superiores.
Cada camada executa alguns serviços para a camada acima dela. A definição do serviço informa o que a camada faz, e não a forma como as entidades acima dela a acessam ou como a camada funciona.
Camadas:
7- Aplicação -> interface com aplicativos: Comunicação maquina-usuario. Browser, software de e-mail etc
6- Apresentação -> Formatos e criptografia: representação dos dados, tradução. Converte o formato do dado recebido da camada de aplicação em um formato comum aser usado na transmissão do mesmo.
5- Sessão -> Sessão entre aplicativos: Responsável pela troca de dados e a comunicação entre hosts. Computadores podem se comunicar, dados são marcados para serem transmitidos.
4- Transporte -> Fim-a-fim, portas de acesso:  receber os dados enviados pela camada de sessão e segmentá-los para que sejam enviados a camada de rede, que por sua vez, transforma esses segmentos em pacotes.
3- Rede -> Roteamente, endereço logico:  procedimento de transferência de comprimento variável de dados de sequências de uma fonte de acolhimento de uma rede para um host de destino numa rede diferente. Roteamento de funções, e também pode realizar a fragmentação e remontagem e os erros de entrega de relatório. Swith
2- Enlace -> Controle de acesso ao meio, endereço fisico: detecta e, opcionalmente, corrige erros que possam acontecer no nível físico. Controle de fluxo e protocolos de comunicação
1- Sinal eletrico, bits: fio de cobre, uma fibra óptica, ondas de rádio.

Arquitetura TCP/IP: Diferente do modelo OSI o conjunto de protocolos são dividos em quatro camadas aplicação, transporte, internet e rede – sendo cada uma responsável pela execução de tarefas distintas, para a garantir a integridade e entrega dos dados trafegados.
Pacotes de dados: Divididos em tamanhos menores
Camadas:
5- Aplicação -> Comunicação entre programas e protoclos de transporte,  agrupa os a padrão da rede.
4- Transporte -> Recebe os dados enviados pela camada de aplicação e transforma em pacotes menores, a serem repassados para a camada de internet. Ela garante que os dados chegarão sem erros e na sequência correta.
3- Rede -> especifica a forma que os dados devem ser encaminhados independentemente do tipo de rede utilizado, fornece datagrama.
1- Fisica -> Especifica a forma que os dados devem ser encaminhados independentemente do tipo de rede utilizado.

Encapsulamento: Os dados atravassam cada uma das camadas citadas acima na maquina amissora. Em cada uma dessas camadas é acrescentada informação ao pacote de dados para garantira transmissão, o cabeçalho. 
Na recepção do pacote em cada passagem pela camadas o cabeçalho é lido e excluido, assim, a mensagem "volta" ao seu estado original.

Oriantado para conexão
